adbook is a tool for building book-like webpages. If you like this demo, you might like adbook, too.

1. Installation

1.1. Rust & adbook

If you have Rust, adbook is available via Crates.io:

$ cargo install adbook # -> $HOME/.cargo/bin

1.2. Ruby & Asciidoctor

You need Ruby and some package manager. I recommend RVM because programs installed with it runed faster than that were with rbenv (somehow).

Then let’s get Asciidoctor gems:

$ rvm install asciidoctor
$ rvm install asciidoctor-diagram

We’re ready!

2. Overview

2.1. Usage

Let’s make a new book:

$ adbook init my-book
Initialized a new adbook project at my-book

And see the files:

$ tree my-book -L 2
tmp
├── book.ron (1)
├── site (2)
└── src (3)
    ├── article.adoc
    ├── index.adoc
    ├── static
    │   └── img
    ├── theme
    │   ├── css
    │   ├── hbs
    │   └── js
    └── toc.ron (4)

8 directories, 4 files
1 book.ron is a base configuration file that maps to BookRon
2 The site directory, where adbook outputs files
3 The source directory, which is basically mapped to the site directory
4 toc.ron is very similar to mod.rs in Rust. They are located in each directory and lists .adoc files in it.

adbook maps files from source directory to site directory, converting .adoc files to html files. To generate html files, Asciidoctor and Handlebars templates are used.

2.2. Differences from mdbook

adbook is highly inspired by mdBook, but it uses AsciiDoc for source files instead of Markdown.

Other differences:

  • adbook uses Prism.js to highlight source files.

  • adbook can’t enjoy Rust playground.

  • adbook doesn’t provide hidden source files (so copy some template to get started).

2.3. Differences from raw asciidoctor command

  • asciidoctor does not create multiple page website.

  • adbook provides some placeholder strings to document attributes, such as {base_url}.
