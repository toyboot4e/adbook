1. Welcome

adbook is a tool for building book-like webpages. If you like this demo, you might like adbook, too.

Scroll right to see a next page; the default theme is using flex layout.

Repository

1.1. Installation

Rust & adbook

If you have Rust, adbook is available via Crates.io:

$ cargo install adbook # -> $HOME/.cargo/bin

Ruby & Asciidoctor

You need Ruby and some package manager. I recommend RVM because programs installed with it ran faster than that were with rbenv (somehow).

Then let’s get Asciidoctor gems:

$ rvm install asciidoctor
$ rvm install asciidoctor-diagram

We’re ready!

2. Overview

2.1. Usage

Let’s create a new book:

$ adbook init my-book
Initialized a new adbook project at my-book

And see the files in it:

$ tree my-book -L 2
tmp
├── book.ron (1)
├── site (2)
└── src (3)
    ├── article.adoc
    ├── index.adoc
    ├── static
    │   └── img
    ├── theme
    │   ├── css
    │   ├── hbs
    │   └── js
    └── toc.ron (4)

8 directories, 4 files
1 book.ron is a base configuration file that maps to BookRon. Ron is a human-friendly file format.
2 site is the site directory, where adbook outputs files.
3 src is the source directory, which is basically mapped to the site directory.
4 toc.ron is very similar to mod.rs and maps to TocRon. They are located in each directory and lists .adoc files in it.

Now we can build our book:

$ cd my-book
$ adbook build

For details, check out the source files of this demo.

2.2. Differences from mdbook

adbook is highly inspired by mdBook, but it uses AsciiDoc for source files instead of Markdown. AsciiDoc is an expressive language, as you can see in the demos.

Other differences:

  • adbook uses Prism.js by default to highlight source code.

  • mdBook supports a bunch more thinkgs:

    • Rust Playground

    • print view

    • search

    • dynamic theme change

2.3. Differences from raw asciidoctor command

  • adbook creates multiple page websites while Asciidoctor creates single page websites by default.

  • adbook provides some placeholder strings to document attributes, such as {base_url}.
